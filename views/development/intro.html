Nanode Development

Welcome to the Developer / Development Portal of Nanode. These pages have been put together to help understand and display how our systems work. These documents have also been created to assist in the future debugging and development of Nanode.

Disclaimer

This documentation will not usually be upto date. Nanode is in constant development and changes frequently. This does mean bugs are prevalent, especially on Mobile. You can view our Incoming section to see what bugs are know, as well as and features planned, or already in development. Bugs cannot currently be reported. This is not a bug.

History

The Past

Nanode Started as a Python program that could read and write text files. It then became a simple HTML document of the same concept. From there my project became an Electron JS program by the name Alpharium, sporting a fair few more features. Returning to a website and learning of SocketIO, Nanode was born.

The Future

As more people work remotely, files get larger, photos more defined, its important to have access. Sharing, collaboration, memories, ideas, logs, being able to save and see what you want, safely, simply and forever is what Nanode aims for. Connectivity to your content is the main goal, in both the UI design and our API and is an important reason for these documents.
